# REST API

Maverick ships a JSON REST API at `/api/v1` on the dashboard server.
Any HTTP client can drive the agent: create + run goals, poll for
results, stream live progress, manage facts and skills, view spend.

The dashboard server is the same FastAPI process that serves the
browser UI; mounting them together keeps the deploy footprint to one
process and one port.

## Auth

By default, no auth -- the server binds to `127.0.0.1` only.

For VPS / remote deployments, set ``MAVERICK_DASHBOARD_TOKEN`` and
bind to `0.0.0.0` behind Caddy (TLS). Every API request then needs
either:

- `Authorization: Bearer <token>` header, or
- `?token=<token>` query parameter

`/healthz` is always open so reverse proxies can probe.

## OpenAPI docs

- Interactive Swagger UI: `http://127.0.0.1:8765/docs`
- ReDoc:                  `http://127.0.0.1:8765/redoc`

## Endpoints

### Goals

| Method | Path | Description |
|---|---|---|
| `POST`   | `/api/v1/goals` | Create + start a goal. Background runs. Returns goal record. |
| `GET`    | `/api/v1/goals` | List goals (filter by `status=`, paginate via `limit=`). |
| `GET`    | `/api/v1/goals/{id}` | Get one goal's full record incl. result. |
| `GET`    | `/api/v1/goals/{id}/events` | Long-poll live progress (`?since=<last_id>`). |
| `POST`   | `/api/v1/goals/{id}/answer` | Answer a queued question to unblock a paused goal. |

### Facts

| Method | Path | Description |
|---|---|---|
| `GET`  | `/api/v1/facts` | All known facts as a `{key: value}` map. |
| `POST` | `/api/v1/facts` | Upsert one fact (`{"key":..., "value":...}`). |

### Skills

| Method | Path | Description |
|---|---|---|
| `GET`    | `/api/v1/skills` | List installed/distilled skills. |
| `POST`   | `/api/v1/skills` | Install from URL / gh:org/repo / local path. |
| `DELETE` | `/api/v1/skills/{name}` | Remove. |

### Spend

| Method | Path | Description |
|---|---|---|
| `GET` | `/api/v1/spend` | Total spend + per-episode breakdown. |

## Example: start a goal from curl

```bash
curl -X POST http://127.0.0.1:8765/api/v1/goals \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $MAVERICK_DASHBOARD_TOKEN" \
  -d '{
    "title": "Compare pgvector and Qdrant",
    "description": "Side by side, write to report.md",
    "max_dollars": 2.0
  }'
```

Response::

```json
{
  "id": 42,
  "status": "pending",
  "title": "Compare pgvector and Qdrant",
  "description": "Side by side, write to report.md",
  "result": null
}
```

Then poll status::

```bash
curl http://127.0.0.1:8765/api/v1/goals/42
```

Or stream live events::

```bash
since=0
while true; do
  resp=$(curl -s "http://127.0.0.1:8765/api/v1/goals/42/events?since=$since")
  echo "$resp" | jq '.events[]'
  since=$(echo "$resp" | jq '.next_id')
  status=$(echo "$resp" | jq -r '.status')
  [ "$status" = "done" ] || [ "$status" = "blocked" ] && break
  sleep 2
done
```

## Example: use a template

```bash
curl -X POST http://127.0.0.1:8765/api/v1/goals \
  -H "Content-Type: application/json" \
  -d '{
    "template": "trip-plan",
    "title": "",
    "params": {"destination": "Lisbon", "duration": "5 days"}
  }'
```

Template-rendered title overrides the empty one.

## Mobile / external app integrations

This API is the backbone for any non-channel frontend:

- **React Native / SwiftUI iOS app** -> bearer token + this API; the
  agent runs on a VPS.
- **Zapier / IFTTT** -> create-goal webhook on schedule.
- **Cron jobs** -> a 6am goal to summarize overnight email + slack.
- **GitHub Actions** -> kick off a Maverick goal on PR open.

Keep ``max_dollars`` and ``max_wall_seconds`` sensible on every call
-- they default to the conservative limits in code, but external
callers should always set explicit caps.
